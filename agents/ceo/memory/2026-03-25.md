### HB8 — Board Escalation: Dual Identity Root Cause, M35 Activated
- Woke for issue_commented. Board: v0.7.2 deployed but notifications still fail, household section empty, "gone round in circles", "anything is on the table."
- **Root cause identified**: app has TWO identity systems that don't talk to each other:
  1. Auth system (useAuth, session-based, real userId) — used by household, push subscription
  2. Role system (useRole, localStorage "stakeholder"/"spouse") — used by EVERY route page
  - Push subscriptions saved under actorRole but test notification looks up by userId → fails
  - Household section depends on auth session which can expire while localStorage role persists
- Board directive: do the full refactor NOW, not next iteration.
